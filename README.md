# Лабораторная работа №11: Консольный чат на Java с сокетами

## Описание проекта
Многопользовательский консольный чат-приложение с использованием TCP-сокетов, реализующее поддержку личных и широковещательных сообщений с логированием через кастомную систему логирования.

## Функциональные возможности
- **Многопользовательский режим** - одновременная поддержка до 10 клиентов
- **Аутентификация** - уникальные никнеймы с проверкой занятости
- **Личные сообщения** - приватная переписка между пользователями
- **Широковещательные сообщения** - рассылка сообщений всем участникам
- **Список пользователей** - отображение всех подключенных клиентов
- **Логирование** - кастомная система логирования в файлы
- **Корректное завершение** - обработка сигналов завершения работы

### Классы и их назначение

#### 1. Server.java
**Назначение:** Основной класс сервера, управляющий подключениями клиентов.
**Основные функции:**
- Запуск и остановка сервера
- Управление пулом потоков для клиентов
- Хранение информации о подключенных клиентах
- Маршрутизация сообщений (личных и широковещательных)
- Логирование системных событий

#### 2. ClientHandler.java
**Назначение:** Обработчик отдельного клиентского подключения.
**Основные функции:**
- Аутентификация пользователя (проверка никнейма)
- Обработка входящих команд от клиента
- Отправка сообщений конкретному клиенту
- Логирование действий пользователя
- Управление жизненным циклом соединения

#### 3. Client.java
**Назначение:** Клиентское приложение для подключения к чату.
**Основные функции:**
- Подключение к серверу
- Чтение и отправка сообщений
- Обработка пользовательского ввода
- Отображение сообщений от сервера
- Грациозное отключение

## Система логирования

### Лог-файлы
При запуске создаются 3 лог-файла в папке `logs/`:

1. **server.log** - общие логи сервера (информация о подключениях, отключениях)
2. **chat_messages.log** - логи всех сообщений чата (личных и широковещательных)
3. **errors.log** - логи ошибок и предупреждений
